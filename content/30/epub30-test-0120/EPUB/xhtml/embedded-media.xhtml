Embedded Media

[OPTIONAL] mo-embed-010 Embedded Media - Playback Flow

Tests whether MO-driven playback of embedded media (audio or video) is supported.

When MO playback reaches the embedded HTML5 audio player below, it should automatically start the referenced audio file, and play it for its full duration (assuming no user interaction takes place, that is to say: no phrase-level navigation, no mouse click-to-start, etc.). Once the embedded audio finishes, MO playback should continue at the next paragraph.

(the audio should say "path resolution ...")

This paragraph follows the embedded HTML5 audio player.

If MO playback correctly controls the start and stop of the embedded audio, the test passes.

[OPTIONAL] mo-embed-020 Embedded Media - Pause / Resume

Tests whether MO-driven pause/resume of embedded media (audio or video) is supported.

When MO playback reaches the embedded HTML5 audio player below, it should automatically start the referenced audio file, and play it for its full duration (assuming no user interaction takes place, that is to say: no phrase-level navigation, no mouse click-to-start, etc.). However, invoking the 'pause' command whilst the embedded audio is active should pause the media, and 'resume' should work as expected as well.

(the audio should say "path resolution ...")

This paragraph follows the embedded HTML5 audio player.

If MO playback correctly controls the start + pause / resume behaviours of the embedded audio, the test passes.
